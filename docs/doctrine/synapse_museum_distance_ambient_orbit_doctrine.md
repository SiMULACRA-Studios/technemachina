# Synapse Museum Distance / Ambient Orbit Doctrine

Status: Visual doctrine  
Applies to: Synapse Map, companion home surface  
Mutation policy: View-only

## Core Principle

The Synapse Map should not load as a crowded interface.

It should load like a curated object of presence: a constellation seen from a respectful distance, slowly moving behind glass or on a museum pedestal.

The first impression should be:

- distant enough to breathe,
- centered,
- elegant,
- slowly alive,
- not frantic,
- not over-zoomed,
- not visually noisy.

## Initial View

On first load, the constellation should appear farther away than the current default.

The user should feel they are viewing the whole artifact first, not being thrown into the middle of the stars.

The default view should prioritize:

- full constellation silhouette,
- negative space,
- centered composition,
- gentle depth,
- slow ambient movement.

## Ambient Motion

The constellation may rotate or drift slowly when idle.

The pace should feel hypnotic in the sense of calm attention, not manipulative trance induction.

The motion should be:

- slow,
- smooth,
- continuous,
- non-jarring,
- easy to interrupt,
- and respectful of user interaction.

User drag, zoom, star selection, or direct map commands should override passive motion.

## Museum Rule

The Synapse Map should feel like a living artifact behind glass.

The interface should support contemplation before control.

## Boundary

This is a view/layout behavior only.

It does not change memory.

It does not change graph records.

It does not mutate source-of-truth state.
