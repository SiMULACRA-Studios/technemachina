# v0.4.6 — Synapse Renderer Registry Online

Status: ONLINE
Date: 2026-06-10
Mutation policy: Read-only / non-mutating renderer architecture

## Summary

Created a Synapse Renderer Registry so renderer selection is no longer hardcoded.

The registry currently exposes:

- canvas: canvas-prototype / contract-ready
- three: threejs-museum-renderer / planned

## Confirmed Browser Test

The Synapse Data Adapter test page reports:

Available Renderers:
- canvas: canvas-prototype / contract-ready
- three: threejs-museum-renderer / planned

## Implemented

Created:

- frontend/synapse/SynapseRendererRegistry.js

Updated:

- frontend/synapse-data-adapter-test.html

Added registry functions:

- listSynapseRenderers()
- getSynapseRenderer(key)
- createSynapseRendererContract(rendererKey, viewModel, state)

## Architectural Meaning

Future renderer switching can happen through a registry instead of direct imports.

Canvas and Three.js can share the same Synapse View Model input contract.

## Boundary

This milestone does not mutate memory.

This milestone does not replace the active canvas renderer.

This milestone does not activate Three.js rendering.

## Doctrine

Adapter prepares the view.

Registry selects the renderer.

Renderer consumes the view.

Memory remains Oracle-gated.
