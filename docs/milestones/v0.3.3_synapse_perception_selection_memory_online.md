# v0.3.3 — Synapse Perception Selection Memory ONLINE

Date: 2026-06-10  
Status: ONLINE  
Scope: Synapse Perception frontend state  
Mutation: Not allowed

## Summary

v0.3.3 adds visible active-card memory to the Synapse Perception panel.

Perception cards now remember which analysis result is currently active in the interface. When a Central Signal or Bridge Candidate card is clicked, the selected card receives a persistent visual highlight until another perception card is selected.

## Confirmed Capabilities

- Central Signal cards can become visually active.
- Bridge Candidate cards can become visually active.
- Selecting a new perception card clears the previous active card.
- Active card state is stored in frontend Synapse state.
- Active card state includes node/source/target metadata when available.
- Map selection still works.
- Bridge path highlighting still works.
- Perception remains read-only.
- No memory mutation, graph rewrite, or backend mutation occurs.

## Design Doctrine

Perception should have focus.

The Daemon should not only show analysis results; it should show which result the Oracle is currently inspecting.

This creates the loop:

Analyze → Select card → Focus star/path → Maintain active perception context

## Interaction Contract Preserved

Click card = select analysis result  
Active card = current perception focus  
Click bridge card = highlight relationship  
Click star = select signal  
Drag = navigate  
Zoom = deliberate  
Blank space = calm  
Analysis = read-only  
Mutation = Oracle-gated

## Result

The Synapse Perception panel now has local selection memory.

The interface can visually maintain the current analysis focus without mutating backend memory.
