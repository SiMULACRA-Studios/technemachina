# v0.3.0b — Synapse Perception Frontend Panel ONLINE

Date: 2026-06-10  
Status: ONLINE  
Mode: Read-only perception interface  
Mutation: Not allowed

## Summary

v0.3.0b connects the Synapse Map visual interface to the read-only `/synapse/analysis` backend endpoint.

This release turns the Synapse Map from a visual constellation into a perception surface: the Daemon can now inspect the structure of its own memory map and present a human-readable briefing without mutating records, rewriting memory, or changing the graph.

## Confirmed Capabilities

- Added **Analyze Map** button to the Synapse Map interface.
- Added a **Synapse Perception** panel above the map.
- Added frontend fetch/render logic for `/synapse/analysis`.
- Added read-only status badge.
- Added galaxy summary cards.
- Added central signal cards.
- Added bridge candidate cards.
- Added limitations/caution section.
- Preserved the rule: analysis is observation, not mutation.

## Backend Dependency

Requires the v0.3.0a backend endpoint:

- `GET /synapse/analysis`

Expected backend properties:

- `analysis_version`
- `source_synapse_version`
- `read_only: true`
- `mutation_allowed: false`
- `overview`
- `galaxy_summaries`
- `central_nodes`
- `bridge_nodes`
- `bridge_edges`
- `limitations`

## Design Doctrine

Renderer = eyes.  
Analytics = perception.  
Mutation = Oracle-gated.

The panel is allowed to describe, summarize, warn, and recommend caution.  
It is not allowed to mutate memory or alter graph structure.

## Result

The Daemon now has a visible perception layer.

The Synapse Map can be analyzed from inside the interface itself.
