# v0.3.4 — Synapse Perception Detail Drawer ONLINE

Date: 2026-06-10  
Status: ONLINE  
Scope: Synapse Perception explanatory interface  
Mutation: Not allowed

## Summary

v0.3.4 adds an Active Perception Detail drawer to the Synapse Perception panel.

Perception cards can now explain themselves. When a Central Signal, Bridge Candidate, or Bridge Relationship card is selected, the interface opens a detail drawer showing structured metadata about why that perception result matters.

## Confirmed Capabilities

- Added Active Perception Detail drawer to the Synapse Perception panel.
- Central Signal cards can populate the drawer.
- Bridge Candidate cards can populate the drawer.
- Bridge Relationship cards can populate the drawer.
- Detail drawer can show type, galaxy, degree, weighted degree, confidence, node ID, source ID, target ID, source label, target label, relation type, reason, and description.
- Active perception card highlighting remains functional.
- Map selection remains functional.
- Bridge path highlighting remains functional.
- Perception remains read-only.
- No memory mutation, graph rewrite, or backend mutation occurs.

## Design Doctrine

Perception should not only identify important structures.

It should explain why they matter.

This release creates the loop:

Analyze → Select card → Focus star/path → Inspect reasoning

## Interaction Contract Preserved

Click card = select analysis result  
Active card = current perception focus  
Detail drawer = explanation layer  
Click bridge card = highlight relationship  
Click star = select signal  
Drag = navigate  
Zoom = deliberate  
Blank space = calm  
Analysis = read-only  
Mutation = Oracle-gated

## Result

The Synapse Perception panel now has an explanatory inspection layer.

The Oracle can inspect not only which signal matters, but why it was surfaced.
