# v0.3.1 — Synapse Perception Inspector Linking ONLINE

Date: 2026-06-10  
Status: ONLINE  
Scope: Synapse Perception frontend interaction  
Mutation: Not allowed

## Summary

v0.3.1 links the Synapse Perception panel to the Synapse Map inspector.

Perception cards are no longer passive text. Central Signal and Bridge Candidate cards can now select related stars on the map, update the lower readout, and redraw/focus the selected signal.

## Confirmed Capabilities

- Central Signal cards can select their corresponding Synapse node.
- Bridge Candidate cards can select a related source or target node.
- Selected nodes update the lower inspector/readout.
- The map redraws after card selection.
- Perception remains read-only.
- No memory mutation, graph rewrite, or backend mutation occurs.

## Design Doctrine

Analysis should not remain trapped in text.

A perception result should be able to point back into the map.

This creates the loop:

Analyze → Inspect → Select → Understand

## Interaction Contract Preserved

Click card = select signal  
Click star = select signal  
Drag = navigate  
Zoom = deliberate  
Blank space = calm  
Analysis = read-only  
Mutation = Oracle-gated

## Result

The Synapse Perception layer is now linked to the visual Synapse Map.

The Daemon can identify important structures and let the Oracle jump directly to them.
