The normal daemon chat can now distinguish between direct browser perception and runtime project awareness.

- `daemon/runtime_context/synapse_frontend_status.json`

- `POST /chat`

- `GET /companion/surface-context`

When asked whether the Synapse Map visual frontend renderer exists, the daemon now correctly responds that:

- it cannot visually perceive the browser canvas directly,
- but runtime project context confirms the Synapse Map frontend prototype exists,
- and active renderer work exists in `frontend/index.html`, `frontend/main.js`, and `frontend/style.css`.
