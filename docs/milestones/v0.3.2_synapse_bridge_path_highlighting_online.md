# v0.3.2 — Synapse Bridge Path Highlighting ONLINE

Date: 2026-06-10  
Status: ONLINE  
Scope: Synapse Perception map-linking interaction  
Mutation: Not allowed

## Summary

v0.3.2 upgrades Synapse Perception Inspector Linking by allowing Bridge Candidate cards to visually highlight the relationship/path between two signals on the Synapse Map.

The perception panel no longer only identifies important bridge candidates in text. It now points back into the constellation and visually marks the bridge relationship.

## Confirmed Capabilities

- Bridge Candidate cards can select a related source or target signal.
- Bridge Candidate cards can store a temporary highlighted bridge path in frontend state.
- The map can render the highlighted bridge path with a distinct gold glow.
- Bridge source/target endpoints receive enhanced visual emphasis.
- Central Signal cards still use normal selected-star focus.
- The lower inspector/readout still updates on linked selection.
- Perception remains read-only.
- No memory mutation, graph rewrite, or backend mutation occurs.

## Design Doctrine

A bridge is not just a node.

A bridge is a relationship that deserves visual emphasis.

This release turns bridge analysis into a navigable visual cue:

Analyze → Find bridge → Click bridge → See relation

## Interaction Contract Preserved

Click card = select signal  
Click bridge card = highlight relationship  
Click star = select signal  
Drag = navigate  
Zoom = deliberate  
Blank space = calm  
Analysis = read-only  
Mutation = Oracle-gated

## Result

The Synapse Map now supports relationship-level perception feedback.

Bridge candidates can be seen directly in the constellation instead of only described in text.
