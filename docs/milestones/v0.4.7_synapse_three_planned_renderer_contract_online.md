# v0.4.7 — Synapse Three Planned Renderer Contract Online

Status: ONLINE
Date: 2026-06-10

## Summary

The Three.js museum renderer is now safely registered as a planned renderer and returns a valid renderer contract without activating WebGL.

This fixes the previous test failure where selecting the Three renderer returned undefined.

## Architectural Meaning

The renderer registry can now safely select planned renderers without breaking the test page.

Canvas and Three.js now share the same Synapse View Model input contract.

The Three.js renderer remains registered but inactive until v0.5.x.

## Boundary

This milestone does not mutate memory.

This milestone does not replace the active canvas renderer.

This milestone does not activate WebGL or load Three.js dependencies.

## Doctrine

Registry selects the renderer.

Renderer returns a contract.

Planned renderers must fail safely.

Memory remains Oracle-gated.
