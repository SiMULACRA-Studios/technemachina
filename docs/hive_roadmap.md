# Project H.I.V.E. Roadmap
## Collaboration Layer for Technemachina Daemon

## Status

H.I.V.E. Status: Documented roadmap umbrella. Not operational. No sharing, syncing, collaboration, or public discovery enabled yet.

H.I.V.E. may be documented now, but it must not become operational until Technemachina Daemon has explicit project identity, visibility modes, permission rules, Synapse graph representation, and governed shared-state boundaries.

## Core Definition

Technemachina Daemon is the local-first intelligence core.

Obsidian Brain is the personal daemon / companion layer inside Technemachina.

The Hub is the future web discovery and publishing surface.

H.I.V.E. is the future permissioned collaboration network powered by Technemachina Daemon.

H.I.V.E. is not a separate product. It is the collaborative ecosystem that becomes possible when local daemons can safely share project-level intelligence by consent.

## Non-Operational Rule

Current Technemachina work remains local-first and private.

H.I.V.E. must not enable:

- public discovery,
- user-to-user collaboration,
- cross-daemon sharing,
- project syncing,
- shared memory,
- shared knowledge graphs,
- profile discovery,
- or collaboration proposals

until the required governance primitives exist.

## Required Primitives Before H.I.V.E. Can Operate

H.I.V.E. requires:

1. Project identity.
2. Visibility modes.
3. Permission rules.
4. Synapse graph representation.
5. Knowledge ingest.
6. Knowledge source registry.
7. Governed memory review.
8. Candidate-to-review flow.
9. Shared-state boundaries.
10. Private-vs-project intelligence separation.

## Dependency Chain

Memory governance
→ Candidate flow
→ Knowledge ingest
→ Synapse graph
→ Visibility modes
→ Hub
→ H.I.V.E. collaboration

## Product Structure

### Local Daemon

The private offline-first core where the user creates, edits, stores, and reasons about work.

### Obsidian Brain

The personal daemon inside Technemachina. It guides, recommends, explains, and collaborates with the user while remaining governed by local memory and review rules.

### The Hub

The future web-facing layer for project pages, posts, topic discovery, research summaries, and invitations.

### H.I.V.E.

The permissioned collaboration network. H.I.V.E. connects projects, not private identities, unless users explicitly choose otherwise.

## Visibility Modes

Future H.I.V.E. project visibility modes:

- Hidden: profile hidden, project may remain private or selectively shared.
- Partial: limited project visibility until access is requested.
- Public: open project visibility.
- Invite-only: collaboration requires approval.
- Secret: excluded from H.I.V.E. entirely.

## Collaboration Rule

H.I.V.E. may merge project-level intelligence only after consent.

It must not merge personal memory, private identity, private daemon state, or unrelated user context.
